# Upcoming Maintenance MVP (встроено в дерево узлов)

## Цель

В MVP отдельный экран/модалка «План обслуживания» не вводится.
Информация о плане обслуживания показывается прямо в существующем дереве узлов на web и Expo.

## Режимы отображения

- **Обычный режим (по умолчанию):** текущее поведение дерева узлов без изменений.
- **Режим плана обслуживания:** дополнительный локальный слой подсказок о сроках обслуживания в тех же карточках узлов.

Переключатель находится рядом с заголовком блока дерева.
Состояние переключателя хранится только локально:

- web: `localStorage` ключ `vehicleDetail.<vehicleId>.nodeMaintenanceMode.enabled`
- Expo: локальное UI-хранилище с тем же ключом

Тот же переключатель доступен и внутри modal/screen поддерева выбранного root-узла.  
Оба переключателя (на основной секции и в modal/screen) управляют **одним и тем же** mode state.

## Top-level + subtree UX

На карточке/странице Vehicle Detail дерево на основном полотне упрощено до **только верхнего уровня**:

- показываются root-узлы в исходном порядке;
- для каждого root: имя, статус (если есть), краткое пояснение (если есть), и компактная plan-сводка в режиме плана;
- клик/тап по root открывает отдельный просмотр его поддерева.

Нижние уровни открываются отдельно:

- **web:** модалка поддерева выбранного root;
- **Expo:** полноэкранная мобильная модалка поддерева.

Отдельный экран «План обслуживания» не создаётся.

## Что показывается в режиме плана

### Листовые узлы

Если у leaf-узла есть meaningful schedule data (`statusExplanation.rule` / `usage` / `lastService`), показываются только доступные строки:

- краткое объяснение (если есть);
- остаток или просрочка по:
  - пробегу;
  - моточасам;
  - дням;
- последний сервис (дата/пробег/моточасы, если есть);
- интервал правила обслуживания (если есть).

Без данных пустые строки не рендерятся.

### Родительские узлы

Для parent-узлов показывается компактная агрегированная сводка по дочерним leaf-узлам:

- `Просрочено: N`
- `Скоро: N`
- `Запланировано: N` (если есть)

Если в поддереве нет узлов с расписанием, сводка не показывается.

В modal/screen поддерева сохраняется прежняя логика действий на узлах и детализация для leaf-узлов.

## Статусы и семантика

- `OVERDUE` → «Просрочено» + величина просрочки, где есть данные.
- `SOON` → «Скоро» + оставшийся ресурс, где есть данные.
- `OK` / future-plan → показывается следующая информация только при наличии расписания.
- `null`-статусы без расписания остаются без шумных «OK»-подписей.

## Что не меняется

- структура и порядок дерева узлов;
- expand/collapse веток;
- бейджи статусов;
- popup «Пояснение расчёта»;
- действия узла:
  - журнал по узлу,
  - добавить сервис,
  - добавить в wishlist,
  - SKU/recommendations/service kits в wishlist flow.

## Shared слой

В `@mototwin/types` и `@mototwin/domain` добавлены типы/хелперы maintenance-plan view model:

- `NodeMaintenancePlanViewModel`
- `NodeMaintenancePlanSummaryViewModel`
- `NodeTreeMaintenanceModeState`
- `buildNodeMaintenancePlanViewModel`
- `buildNodeMaintenancePlanSummary`
- `getNodeMaintenancePlanShortText`
- `getNodeMaintenanceDueText`

Хелперы pure и framework-agnostic.
