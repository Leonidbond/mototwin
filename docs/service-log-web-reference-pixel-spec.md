# Спецификация веб-журнала обслуживания по референсу

**Источник правды:** `images/examples/service-log-web.png`  
**Назначение:** единый документ для реализации UI «в точности по макету» — размеры, отступы, сетка, типографика, цвета, иконки, линии, состояния.  
**Связанные документы:** `docs/mototwin_service_log_ui_spec_web_mobile_v_1.md` (продуктовая логика и mobile), `docs/service-log-mvp.md`.

---

## 0. Метод анализа и точность

- Референс — растровый PNG; **абсолютные пиксели** без Figma/Sketch подтверждаются визуально и через инспектор в браузере после наложения скриншота.
- Ниже заданы **целевые значения** и **допустимые отклонения ±1–2 px** для типографики и радиусов.
- Где в разных проходах анализа расходились детали (например, жёлтый vs оранжевый бейдж у «Фильтров»), в спецификации указано **приоритетное требование по последнему разбору референса** и пометка «проверить по PNG».

## 1. Общая композиция (desktop)

| Зона | Поведение |
|------|-----------|
| Левая колонка — список журнала | `minmax(0, 1fr)` в CSS-grid; `width: 100%`, `min-width: 0`; вертикальный скролл всей страницы |
| Правая колонка — детали события | Фиксированные **460px** (`SERVICE_LOG_DETAILS_COL_WIDTH`); **`position: sticky`**, `top: 16px`, `max-height: calc(100vh - 32px)` |
| Горизонтальный `gap` между колонками | **16px** (только на широком viewport ≥ 1180px) |

**Контентная область:** горизонтальные отступы от края сайдбара до контента **24px** (`padding-left/right` секции контента).  
**Ширина контента:** **на всю доступную ширину** под сайдбаром (`max-width` снят — раньше был 1500px, теперь убран, см. §14). Центрирование `margin: 0 auto` не применяется.

**Фон страницы (за карточками):** ровный токен `productSemanticColors.canvas` (= `#080D12`), как в гараже. Вертикальный градиент намеренно убран — он ломал визуальную преемственность с другими страницами (см. §14).

## 2. Цветовая система (референс)

> **Источник правды — `productSemanticColors` из `@mototwin/design-tokens`** (значения откалиброваны по `images/examples/garage web.png`). Журнал не должен заводить параллельную палитру: все цвета этой страницы обязаны мапиться в токены. Старые hex ниже сохранены только как комментарий «откуда взялись числа в спеке», но в коде их быть не должно.

|------|-----------------------------------|------------|------------------------------------|

| `bg.control` | `cardSubtle` | Поиск, дропдауны, вторичные контролы | `#0B1018` |

| `border.strong` | `borderStrong` | Утолщённая обводка/divider | — |
| `text.primary` | `textPrimary` | Заголовки, значения | `#F8FAFC` |
| `text.secondary` | `textSecondary` | Подписи строк, «Найдено», лейблы дропдаунов uppercase | `#94A3B8` |
| `text.muted` | `textMuted` | Placeholder поиска, VIN, второстепенные подписи | `#64748B` / `#8B95A8` |
| `accent.primary` | `primaryAction` | Primary CTA, активный switch ON, выделение выбранной строки, бейдж «Фильтры» | `#FF6B00` |
| `on.accent.primary` | `onPrimaryAction` | Текст/иконка поверх `accent.primary` | `#FFFFFF` |
| `status.success` | `successStrong` (текст), `successSurface` + `successBorder` (плашки) | Бейдж «Выполнено», стоимость в зелёном | `#22c55e` |

## 4. Верхний блок страницы (вне карточки журнала)

### 4.1 Структура строки

Одна горизонтальная полоса: **`display: flex; justify-content: space-between; align-items: center`**.

- **Слева:** `[Назад]` + блок текста (название + VIN).
- **Справа:** одна primary-кнопка **«Добавить ТО»** (открывает тот же сценарий создания, что и раньше вторая кнопка в паре; ghost **«Добавить событие»** не показывается).

**Вертикальные отступы блока:** `padding: 18px 0 22px` (без дополнительного горизонтального padding — он задаётся контейнером **24px**).

### 4.2 Кнопка «Назад»

- **Размер hit-area:** **40×40px**.
- **Фон:** прозрачный; **без** обводки вокруг квадрата на референсе.
- **Иконка:** шеврон влево, stroke **2px**, размер визуально **~20–22px**, цвет **`#F1F5F9`**.

### 4.3 Название и VIN

- Название: см. таблицу §3.
- Отступ между названием и VIN: **4px**.
- Формат VIN: строка `VIN: {номер}`.

### 4.4 Кнопка «+ Добавить ТО» (primary)

Используется `<Button variant="primary">` — поверхность и цвет берутся из `productSemanticColors.primaryAction` / `onPrimaryAction` (тот же оранжевый, что у CTA «Открыть» в гараже).

- **Высота:** **40px**, **радиус 8px** (`style`-overrides).
- **Фон / Текст:** **через токены** (`primaryAction` / `onPrimaryAction`) — не вписывать `#FF6B00` локально.
- **Иконка «+»** слева (`leadingIcon={<PlusSmallSvg />}`), **~15px**, stroke **~2.2px**.
- **Тень:** не задаём — `Button` сам не рисует `box-shadow`.

## 5. Левая колонка журнала

### 5.1 Контейнер

В текущей реализации внешняя «карточка-обёртка» вокруг тулбара/инфо-строки/списка месяцев **намеренно снята** — она съедала ширину и дублировала визуальную рамку с карточками месяцев. Левая колонка — это flex-column (`journalCardLayoutStyle`) **без** `border` / `background` / `border-radius`. Поверхностью отдельных блоков управляют:

- **Тулбар** — без фона, только `border-bottom` (см. §6).
- **Инфо-строка «Найдено / Сортировка»** — без фона, `border-bottom` (см. §7).
- **Список месяцев** — каждая месячная секция рисует собственную «вложенную карточку» (см. §8).
- **Правая sticky-карточка деталей** сохраняет внешнюю поверхность с `radiusScale.xl` (см. §10).

Если потребуется вернуть единую внешнюю карточку — заводить через `<Card padding="none" radius="xl">` из `src/components/ui/Card.tsx`. Не делать локальный `<div>` со своими hex-значениями.

### 5.2 Шапка страницы (выше тулбара)

Шапка с кнопкой «Назад», названием/VIN и primary **«Добавить ТО»** — отдельный блок над тулбаром (см. §4). Padding: `18px 0 22px` (вертикальный), горизонтальные нули — выровнено по краю колонки.

### 5.3 Разделители

- Под тулбаром — `border-bottom` 1px `border.subtle`.
- Под инфо-строкой — `border-bottom` 1px `border.subtle`.
- Между шапкой и тулбаром — только воздух.

## 6. Панель фильтров (toolbar)

### 6.1 Общие параметры

- **`display: flex; flex-wrap: nowrap; align-items: stretch`**.
- **`gap`:** **10px** (`REF_TOP.toolbarGap`).
- **`padding`:** **`16px 0 18px`** — горизонтальные нули, тулбар выровнен **по краю колонки** (тот же край, что и у карточек месяцев и инфо-строки). Тогда поле поиска и кнопка «Фильтры» доходят до самой границы списка событий, без двойного отступа.
- **`width: 100%`**, **`box-sizing: border-box`**, **`min-width: 0`**.
- **`border-bottom`:** `1px` `border.subtle`.

### 6.2 Поле поиска

- **Высота:** **44px** (`REF_TOP.searchH`).
- **Радиус:** **9px** (`TOOLBAR_REF.moduleRadius`, в диапазоне 8–10 по спеке).
- **Фон:** `bg.control` (`productSemanticColors.cardSubtle`).
- **Border:** `border.subtle` (как у остальных модулей тулбара).
- **Padding горизонтальный:** **10px**; **gap** между иконкой лупы и текстом **8px**.
- **Иконка лупы:** **16×16px**, stroke **2px**, цвет как placeholder (`text.muted`).
- **Текст ввода:** **14px**, цвет `text.primary`.
- **Placeholder:** «Поиск по событию, узлу, комментарию» — цвет `text.muted` (через `::placeholder`).
- **Гибкость:** `flex: 2.4 1 0%`, `min-width: 160px` — забирает большую долю свободного места (~2.4 от каждой 1× у дропдаунов).

### 6.3 Модули «Тип события», «Узел», «Период»

- **Высота:** **44px** (ровно как поиск).
- **Гибкость:** `flex: 1 1 0`, `min-width: 88px`. Жёстких `max-width` **не задаём** — поля делят оставшуюся ширину пропорционально.
- **Двухстрочная разметка:** сверху **uppercase** лейбл **9–10px** / `text.secondary`; снизу значение **12–13px** / `text.primary` / **600**.
- **Chevron** справа (где применимо): **9–11px**, отступ справа **5px**, цвет `text.secondary`, **`pointer-events: none`**.
- **Тип события:** кликабельный `<select>` на всю область (оверлей как раньше).
- **Узел:** кнопка открывает модалку выбора листьев (`NodePickerModal`); значение по умолчанию — «Все узлы»; при активном фильтре — подпись из URL (`nodeLabel`).
- **Период:** кнопка открывает **панель** под полем (пресеты + два видимых `type="date"`). У обёртки периода задано **`overflow: visible`**, иначе панель режется общим `overflow: hidden` у базового стиля дропдауна.

### 6.4 Toggle «Только оплаченные»

- **Track:** **36×20px**, `border-radius: 999px`.
- **OFF:** фон **`rgba(148,163,184,0.28)`** (тёмно-серый трек).
- **ON:** фон **`accent.primary`**.
- **Thumb:** круг **16px**, белый, вертикально по центру трека; **OFF** `left: 2px`, **ON** `left: 36 - 16 - 2 = 18px`.
- **Лейбл** справа от трека: **13px**, отступ от трека **10px**, цвет **`#CBD5E1`**.

### 6.5 Кнопка «Фильтры» и раскрываемая строка

- **Кнопка «Фильтры»:** высота **44px**, тот же фон/бордер что у полей; иконка фильтра слева (**13px**, stroke **2px**); текст «Фильтры» **12px** / **500**; chevron раскрытия **▾/▴**; **бейдж счётчика** (если > 0): оранжевый круг (`primaryAction` / `onPrimaryAction`), мин. **18px**, цифра **10px** / **800**.
- **Раскрытая строка** (под тулбаром, `border-bottom` как у тулбара): одна горизонтальная полоса, **компактные** поля (~**30px** высота, радиус **6px**, шрифт **11–12px**): группа **Пробег** (от — до), **Сумма** (от — до), селекты **Тип** / **Исп.**; справа кнопка **«Сброс»** (`margin-left: auto` при широкой колонке). При нехватке ширины — **`overflow-x: auto`** у контейнера строки.

## 7. Строка «Найдено» и сортировка

- **`padding`:** **`6px 24px 8px`** (`REF_TOP.infoPad`). Горизонтальный 24px — **технический отступ для текста**, чтобы «Найдено: 36 событий» не лип к самому краю колонки. Сама полоса (`border-bottom`, `width: 100%`) идёт от края до края.
- **`border-bottom`:** `1px` `border.subtle`.
- **Слева:** «Найдено: **N** событий» — префикс **400** / secondary, число **600** / primary; опционально чип сброса фильтра по узлу из URL.
- **Справа:** единый контрол сортировки:
  - Высота **28–32px** (`REF_TOP.sortComboH`, текущее значение **28**), радиус **8px** (`REF_TOP.sortComboRadius`), фон/бордер как у полей.
  - Внутри: префикс «Сортировка:» (**400**, secondary) + значение (**500**, primary).
  - Chevron справа, `padding-right: 30px` для текста.
- Переключатели «список / сетка» после сортировки **удалены** (не использовались).

## 9. Строка события в списке (timeline row)

### 9.1 Сетка колонок

Колонки заданы через CSS-grid в **долях `fr`** (а не фиксированных px), чтобы лишняя ширина уходила в заголовок и метрики. Текущее значение `gridTemplateColumns`:

```
minmax(72px, 1.1fr) 32px 44px minmax(0, 2.5fr) minmax(88px, 1fr) minmax(88px, 1fr) minmax(96px, 1fr) 22px
```

| # | Зона | Ширина | Содержимое |
|---|------|--------|------------|
| 1 | Дата / пробег | `minmax(72px, 1.1fr)` | См. §9.4 — три строки: «14 апр.» / «2026» / «42 180 км» |
| 2 | Rail + точка | `32px` | Вертикальная линия (1px), центрированная; круг **16px** на линии (см. §9.2) |
| 3 | Иконка типа работ | `44px` | Круг **36px** с цветным фоном по типу + SVG **13px** (см. §9.6) |
| 4 | Заголовок | `minmax(0, 2.5fr)` | Title + subtitle, **ellipsis**; забирает большую часть свободного места |
| 5 | Стоимость | `minmax(88px, 1fr)` | Иконка **монета с ₽** + значение **12.5px**; подпись CAPS **9.5px** (см. §9.5) |
| 6 | Интервал | `minmax(88px, 1fr)` | Иконка часов + значение/лейбл аналогично |
| 7 | Исполнитель | `minmax(96px, 1fr)` | Иконка человека + значение/лейбл аналогично |
| 8 | Chevron | `22px` | Символ **›**, **16px**, цвет secondary / accent если selected |

- **`gap`** между колонками: **12px**.
- **Padding строки:** **`10px 24px`** — горизонтальные 24px, ровно как у шапки месяца и инфо-строки. Тогда первая колонка («14 апр.») визуально выровнена по тексту «Найдено» вверху.
- **Разделитель между строками:** `border-top: 1px` `border.subtle` начиная со **2-й** строки в группе.
- **Фон выбранной строки:** мягкий оранжевый wash `${primaryAction}10` (`SPEC.rowSelectWash`, ≈6% alpha).

### 9.2 Вертикальная линия таймлайна

- **Ширина:** **1px**.
- **Цвет:** приглушённый серый (`SPEC.timelineLine = rgba(148,163,184,0.35)`).
- **Высота:** от **50%** высоты строки (первая в группе) или **0** (верх) до **50%** (последняя) — «леска» между узлами.
- **Точка на линии:** круг **16px**; для выбранной — заливка `accent.primary` + белая галочка; для сервиса — обводка зелёная + зелёная галочка; для STATE_UPDATE — обводка muted.

### 9.3 Иконки в метриках строки

- **Размер:** **13×13px**, stroke **2**, `stroke-linecap/linejoin: round`.
- **Цвет иконки:** `text.secondary` (`#94A3B8` через токен `productSemanticColors.textSecondary`), не ярче значения.

### 9.4 Колонка даты (3 строки)

Колонка 1 рендерится через page-local хелпер `formatRowDateParts(iso)`:

| Строка | Содержимое | Стиль |
|--------|-----------|-------|
| 1 | День + месяц (`14 апр.`) | **13px / 700**, `text.primary`, `line-height: 1.2`, ellipsis |
| 2 | Год (`2026`) | **13px / 700**, `text.primary`, `line-height: 1.2` |
| 3 | Пробег (`42 180 км`) | **11px / 400**, `text.muted`, `margin-top: 2px` |

> Источник пробега в строке — `entry.odometerValue` (значение, например `«42 180 км»`), **не** `odometerLabel` (это слово «Пробег»). Это распространённая ошибка, исправлено явно.

### 9.5 Иконка «Стоимость» — монета с ₽

В колонке 5 слева от значения стоит **`<CoinSvg />`** (монета с символом ₽), а не гаечный ключ. Подпись CAPS — «Стоимость», значение зелёное при наличии суммы (`successStrong`), `«—»` muted при отсутствии.

### 9.6 Иконка типа работ (Col 3)

Иконка повторения / двойных стрелок (бывш. `ReplaceSvg`) **полностью убрана**. Маппинг теперь такой:

| `actionType` | Иконка |
|--------------|--------|
| `SERVICE` | гаечный ключ (`WrenchSvg`) |
| `REPLACE` | гаечный ключ (`WrenchSvg`) — единая иконка обслуживания |
| `INSPECT` | лупа (`InspectSvg`) |
| `STATE_UPDATE` | солнце-state (`StateSvg`) |

Заливка/цвет круга — из `getServiceIconConfig(actionType)`.

## 10. Правая панель деталей

- **Внешняя карточка:** **`border-radius: 18px`**, обводка как журнал, фон `bg.panel`.
- **`sticky`**, **`top: 16px`**, **`max-height: calc(100vh - 32px)`**, колонка **`flex`** с **`min-height: 0`** и скроллом тела.
- **Шапка:** иконка типа **40px**, заголовок, подзаголовок, крестик закрытия; бейдж **«Выполнено»** — зелёный pill.
- **Метрики:** одна строка, **4 колонки** равной ширины; **лейбл сверху** (CAPS 10px muted), **значение снизу** (13–15px bold); вертикальные разделители **optional** `1px` между колонками как на референсе.
- **Секции:** «Установленные запчасти» (+ счётчик), строки с **плейсхолдером 48×48** или миниатюрой, цена справа; «Комментарий»; «Исполнитель» (аватар + имя); «Источники».
- **Нижняя панель действий:** три кнопки **outline** на всю ширину, **Редактировать** / **Удалить** (красный акцент) / **Повторить ТО**; **`border-top: 1px`**, фон совпадает с панелью.

- Вторичная кнопка: высота **~40–44px**, горизонтальный padding **22px**, радиус **10px**, обводка subtle, текст **12–13px** / **600**, иконка/стрелка **▼** после текста.

## 12. Иконки — общие правила

- Стиль: **outline**, **stroke 2px** (для мелких **1.5–2px**).
- Размеры в UI: **13–16px** в строках и тулбаре; **18–20px** в плейсхолдере запчасти.
- Цвет по умолчанию: **`currentColor`** или привязка к `text.secondary`.
