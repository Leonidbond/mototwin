# Страница «Узлы» — функционал (Mobile + Web)

Документ фиксирует текущее поведение страницы состояния узлов для мобильного и веб-интерфейсов MotoTwin.

---

## 1) Назначение страницы

Страница «Узлы» нужна для:

- просмотра состояния узлов мотоцикла (OK / SOON / OVERDUE / UNKNOWN);
- поиска и фильтрации узлов;
- переходов к связанным действиям (журнал, расходы, ТО, подбор деталей);
- работы с контекстом узла (рекомендации, сервис-киты, пояснения статуса);
- контроля расходов по узлу и поддереву.

---

## 2) Роуты

### Mobile (Expo)

- основной экран состояния узлов: `/vehicles/[id]/nodes`
- реализация: `apps/app/app/vehicles/[id]/index.tsx` (режим `nodes`)

### Web (Next.js)

- экран деталей мотоцикла с секцией узлов: `/vehicles/[id]`
- отдельная страница полного дерева: `/vehicles/[id]/nodes`
- рендер/логика узлов: `src/app/vehicles/[id]/vehicle-detail-client.tsx`

---

## 3) Общая функциональность (и mobile, и web)

- загрузка дерева узлов мотоцикла;
- построение view-model для иерархии и статусов;
- фильтр по статусу узла;
- поиск по узлам;
- раскрытие/сворачивание веток дерева;
- подсветка/фокус выбранных узлов;
- открытие контекста узла (действия и доп.информация);
- учет расходов по узлам за выбранный сезон;
- переход в журнал обслуживания с фильтром по узлу;
- переход в расходы с фильтром по узлу/поддереву.

---

## 4) Mobile: функционал страницы `/vehicles/[id]/nodes`

### 4.1 Верхняя часть экрана

- заголовок «Состояние узлов»;
- переключатель режима обслуживания;
- кнопка перехода в журнал обслуживания;
- краткое описание секции.

### 4.2 Управление деревом

- поле «Поиск по узлам»;
- фильтр по статусам (чипы, включая «Все» и счетчики по статусам);
- выбор сезона расходов по узлам;
- подсказки/сообщения об ошибках загрузки сводки расходов по узлам.

### 4.3 Дерево узлов

Для каждого узла:

- название, путь/код и индикатор статуса;
- раскрытие/сворачивание дочерних узлов;
- короткие рекомендации/причина статуса;
- действия:
  - открыть контекст узла,
  - добавить сервисное событие,
  - открыть журнал по узлу,
  - открыть расходы по узлу.

### 4.4 Контекст узла (modal)

- показывается расширенная информация по узлу;
- доступны быстрые действия (журнал, ТО, wishlist, kits и т.д.);
- возможно открытие пояснения статуса;
- поддерживаются действия по snooze (где применимо).

---

## 5) Web: функционал секции узлов на `/vehicles/[id]`

### 5.1 Обзор узлов

- карточки верхнеуровневых групп/узлов;
- быстрый drill-down в проблемные узлы;
- статусы и агрегированные показатели по группам.

### 5.2 Полное дерево

- показ иерархии узлов с состояниями;
- компактный поиск по узлам;
- фильтрация по статусам вынесена отдельной строкой под поиском;
- переключатель «ТОП-узлы» оставляет до 15 узлов из блока «Состояния узлов» и цепочки их родителей до корня;
- кнопка «Свернуть дерево» закрывает все раскрытые ветки;
- раскрытие веток; если у узла ровно один дочерний узел, цепочка раскрывается автоматически до первого ветвления или листа;
- выделение узла и отображение связанных данных.

### 5.3 Контекст и действия

- открытие деталей узла в боковом/модальном контексте;
- левая колонка дерева и правая панель контекста прокручиваются независимо;
- для leaf-узлов не показывается блок «Что входит в узел»;
- для узлов верхнего уровня блок «Что входит в узел» показывает полный состав поддерева в строку с разделителями, поддерживает раскрытие длинного списка;
- элементы в блоке состава кликабельны и переводят фокус на соответствующий узел дерева; прямые дочерние узлы выделяются жирным;
- переходы в журнал/расходы/действия обслуживания;
- сценарии добавления связанных сущностей (service event, wishlist и др.).

### 5.4 Расходы по узлам

- выбор сезона расходов;
- вывод сумм по узлу и поддереву;
- переход на `/expenses` с параметрами узла для детальной аналитики.

---

## 6) Правила поведения расходов по узлам

- parent-узел агрегирует расходы своего `nodeId` + всех дочерних узлов;
- leaf-узел показывает только собственные расходы;
- сезон расходов = календарный год;
- разные валюты отображаются раздельно (без конвертации);
- фильтр по `nodeId` на странице расходов трактуется как узел + поддерево.
