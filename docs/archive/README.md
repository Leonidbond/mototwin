# Архив документации (`docs/archive`)

Сюда перенесены материалы, которые **не являются** актуальным source of truth для разработки, но могут пригодиться как контекст: старые планы миграции, разовые аудиты, пошаговые заметки по отдельным экранам Expo и черновики продуктовых идей.

## Содержимое

| Файл / группа | Зачем хранить |
|---------------|----------------|
| `expo-bootstrap.md`, `expo-first-migration-plan.md` | Ранний план перехода на Expo |
| `documentation-gap-analysis.md` | Снимок пробелов в доках (частично закрыт текущим `docs/README.md`) |
| `project.md`, `node-status.md` | Исторические продуктовые / статусные заметки; актуальная логика статусов — в `functional-logic.md` |
| `expo-screen-*.md` | Пошаговые заметки по экранам; маршруты и поведение — в `frontend-expo.md` и соответствующих MVP-спеках |
| `expenses-analytics-vision-draft.md` | Черновик идеи страницы «Расходы» (бывший `expences.md`); операционный MVP — `expense-tracking-mvp.md` |

## Правило использования

Перед изменением кода ориентируйся на **корневые** документы из раздела «Canonical» / «Feature specs» в [`../README.md`](../README.md). Архив не обновляется синхронно с кодом.
