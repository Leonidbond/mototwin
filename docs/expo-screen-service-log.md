# Expo Screen: Service Log

## What was built

A Service Log screen (`apps/app/app/vehicles/[id]/service-log.tsx`) accessible from the Vehicle Detail screen via the "Журнал обслуживания" button.

Events are grouped by month (newest first), matching the web journal structure.

## Navigation

```
Garage → Vehicle Detail → [Журнал обслуживания] → Service Log
```

Vehicle Detail is now at `apps/app/app/vehicles/[id]/index.tsx` (moved from `[id].tsx` to support the nested route).

## Route/query params

Supported params for Expo route:

- `paidOnly=1|true` - pre-enables paid-only filter
- `nodeIds=<id1,id2,...>` with optional `nodeLabel` - pre-applies subtree filter
- `serviceEventId=<eventId>` (alias: `highlightServiceEventId`) - highlights matching row
- `feedback=created|updated` - shows action notice after returning from add/edit flow
- `expandExpenses=1|true` and `month=<YYYY-MM>` are accepted in route builder for deep links/handoffs

## Data

Uses `createMotoTwinEndpoints(createApiClient(...))` from `@mototwin/api-client`.

Key requests on screen load:

- `GET /api/vehicles/:id/service-events`
- `GET /api/vehicles/:id` (title/odometer/engine hours)

## Grouping and summaries

Uses shared helpers from `@mototwin/domain`:

- `buildServiceLogTimelineProps` - filters, sorts and groups timeline into month groups
- `getServiceLogEventKindBadgeLabel` - badge labels by event kind
- `filterPaidServiceEvents` - paid-only helper labels/empty-state logic
- `isServiceLogTimelineQueryActive` - query-state indicator

## Event rendering

### SERVICE events
- Kind badge: "Сервис" (purple)
- Title: `serviceType`
- Secondary: node name
- Meta: date · odometer · engine hours
- Optional: comment, cost
- Optional: linked part (`partName`/`partSku`)
- Row actions: repeat, edit, delete
- Wishlist-origin label/button is rendered for applicable rows (`installedPartsJson`)

### STATE_UPDATE events  
- Kind badge: "Состояние" (grey)
- Title/description generated by shared domain timeline view models
- Meta: date · odometer · engine hours
- No edit/delete actions

## Month header

Each month shows:
- Month + year label (e.g. "Январь 2024")
- Summary: service count, state update count, total cost per currency

## Filters and sorting

Expo Service Log supports:

- filters: `dateFrom`, `dateTo`, `eventKind`, `node`, `serviceType`
- route-level subtree filter chip (`nodeIds/nodeLabel`) with quick clear
- paid-only toggle (`paidOnly`)
- sorting by field and direction

## States

- Loading spinner
- Error with message
- Empty: shown when no events exist yet
- Populated: grouped list
- Action notice after create/update/delete

## Lifecycle actions

- **Create/edit/repeat**: opens `/vehicles/[id]/service-events/new` with query params:
  - `source=service-log`
  - `eventId=<id>` for edit
  - `repeatFrom=<id>` for repeat
- **Delete**: calls `DELETE /api/vehicles/[id]/service-events/[eventId]` for `SERVICE` rows only
- **Expenses entry point**: header action opens `/vehicles/[id]/expenses`

## Shared packages used

- `@mototwin/types` - timeline and filter types
- `@mototwin/domain` - timeline builder and query helpers
- `@mototwin/api-client` - service events and mutation requests
