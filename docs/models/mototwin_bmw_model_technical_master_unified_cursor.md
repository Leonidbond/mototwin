## 1. Что исправлено

Предыдущий BMW-файл уже содержал технические параметры, но структура полей отличалась от будущего общего стандарта: например, использовалось `power_hp` вместо универсальной пары `power_value + power_unit`.

Этот файл приведен к единой структуре:

```text
Brand → ModelFamily → Variant → Generation → TechnicalSpecs
```

Эта же структура должна использоваться для KTM, Honda, Yamaha, Kawasaki, Ducati, Triumph и других брендов.

## 4. Prompt для Cursor

```text
Используй этот файл как BMW seed для MotoTwin Model Technical Master.

Требования:
1. Импортируй BMW по общей структуре Brand → ModelFamily → Variant → Generation → TechnicalSpecs.
2. Не используй старое поле power_hp как отдельный стандарт. Используй power_value + power_unit + power_hp_normalized.
3. Пустые значения из CSV преобразуй в null.
4. Каталог запчастей должен ссылаться на generation_id.
5. Не создавай отдельную BMW-схему. BMW, KTM и остальные бренды должны использовать один importer и одну Prisma-модель.
6. После импорта проверь счетчики:
   - Brand: 1
   - Model families: 10
   - Variants/configurations: 14
   - Generations: 14
```
