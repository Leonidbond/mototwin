# Шаблон находки и шкала severity

## Шкала severity

| Severity | Когда применять | SLA (бета) |
|----------|-----------------|------------|
| **P0** | Удаленное выполнение кода, обход аутентификации, массовое раскрытие PII, чтение/запись чужих ресурсов без аутентификации, утечка приватных ключей в продакшене, угон аккаунта при типичной атаке без условий. | Фикс до публичного беты-релиза; релиз заморожен до фикса. |
| **P1** | Privilege escalation в пределах одного пользователя, account takeover при «удобной» цепочке (например, отсутствие audience-проверки OAuth), отсутствие критичной защиты (rate limit на login/forgot/oauth, CSP/HSTS, audit-log), account enumeration на критичных endpoint-ах. | Фикс в течение текущего спринта. |
| **P2** | Defense-in-depth, не эксплуатируемое в текущей модели угроз, hardening cookie-флагов, дополнительный аудит-лог, MASVS L2-элементы, документация. | В техдолг, фикс по плану. |

## Шаблон записи находки

```
### MT-SEC-XXX <короткое название>

- **Severity:** P0 / P1 / P2
- **OWASP:** API1 / A05 / M3 / …
- **Scope:** api / web / mobile / infra
- **Status:** open / in-progress / resolved / risk-accepted
- **Owner:** <команда или ФИО>
- **Effort:** S (≤1ч) / M (≤1 дня) / L (>1 дня) / XL (>3 дней)

**Evidence:**
- `src/path/file.ts:42-58`
  ```
  <минимальный фрагмент кода>
  ```

**Impact:**
<кто и как может это эксплуатировать; какие активы под угрозой>

**Likelihood:**
<насколько реалистична эксплуатация в текущей модели угроз>

**Recommendation:**
<что и как поправить, в идеале — диффер или sketch>

**Acceptance criteria:**
- [ ] <проверяемое утверждение 1>
- [ ] <тест / код-проверка 2>
- [ ] Обновлен/добавлен тест, который красный без фикса и зеленый с фиксом.
```

## ID-нумерация

- Префикс `MT-SEC-` + 3-значный счетчик, начиная с `001`.
- ID присваивается один раз и не переиспользуется (даже если находку закрыли).
- При дублировании — закрыть позднюю как `duplicate of MT-SEC-XXX`.

## Связь между документами

- `findings.md` — единый реестр (только короткое описание + severity + ссылка).
- `api-findings.md` / `web-findings.md` / `mobile-findings.md` — детальный разбор по категориям OWASP с цитированием кода.
- `roadmap.md` — план фиксов, сгруппированный по приоритету, с acceptance criteria.
