# Parts catalog QA seed (`parts-skus.qa.json`)

## Назначение

Файл `prisma/seed-data/parts-skus.qa.json` содержит **демонстрационные** SKU для ручного QA на четырёх мотоциклах из основного сида:

- BMW F 850 GS 2022  
- BMW R 1250 GS 2023  
- KTM 690 Enduro R 2022  
- KTM 890 Adventure 2023  

Данные **не** являются полным OEM/aftermarket каталогом. Цены, офферы и формулировки применимости — **только для стенда и тестов**.

## Как загружается

1. В `prisma/seed.ts` после сидов **User**, **Brand**, **Model**, **ModelVariant**, **Node** и после базового каталога `parts-skus.json` вызывается загрузчик QA JSON.
2. `PartSku` находится или создаётся по стабильному ключу: **`brandName` + `canonicalName` + `partType`** (trim, без отдельного уникального `seedKey` в JSON).
3. `PartNumber`: нормализация для QA — trim, upper case, удаление пробелов и **дефисов**; уникальность в БД: `(skuId, normalizedNumber, numberType)`.
4. `PartSkuNodeLink`: узел резолвится по **`nodeCode`**; если узла нет — связь пропускается, сид **не падает**.
5. `PartFitment`: в JSON задаются **`brandName` / `modelName` / `modelVariantName`** (и при необходимости годы); сид резолвит их в `Brand` / `Model` / `ModelVariant`. Если модель или модификация не найдены — пишется предупреждение, по возможности создаётся более общая строка (например, только бренд) или запись пропускается.
6. `PartOffer`: для источника **QA Seed** используется стабильный **`externalOfferId`** (детерминированный хеш от SKU-ключа и заголовка оффера), чтобы **повторный прогон сида не плодил дубликаты**.

## Демо-мотоциклы в гараже

После сида у пользователя **`demo@mototwin.local`** появляются два стабильных мотоцикла (идемпотентно по `nickname`):

- **QA — BMW R 1250 GS 2023**
- **QA — KTM 690 Enduro R 2022**

Они нужны для ручной проверки **рекомендаций по узлу** и совпадения `PartFitment` с `parts-skus.qa.json`. Профиль поездки (`RideProfile`) создаётся/обновляется автоматически.

## Запуск

Из корня репозитория (нужен `DATABASE_URL`):

```bash
npx prisma generate
npm run db:seed
```

В этом репозитории сид подключается скриптом **`tsx prisma/seed.ts`** (`package.json` → `db:seed`). Команда `npx prisma db seed` сработает только после настройки `seed` в `prisma.config.ts` (сейчас не обязательна для работы).

Повторный запуск **безопасен**: те же SKU и связанные строки обновляются, а не дублируются (в пределах описанных правил уникальности).

## Ограничения и честность данных

- **Цены** в JSON — условные QA-значения, не рыночные и не «живые» прайсы.
- **Метки совместимости** в описаниях/fitment отражают сценарий теста; продуктовые рекомендации в приложении остаются **консервативными** (см. `docs/parts-catalog-mvp.md`).

## Проверки после сида

- `GET /api/parts/skus?search=Motul` — находятся QA SKU Motul / Motorex и т.д.  
- Поиск по артикулу после нормализации: например `HF155`, `DID`, `Yuasa`.  
- Рекомендации по узлу и wishlist с привязкой к SKU продолжают работать на загруженных данных.
