# MotoTwin — Подробное описание интерфейса окна сервисного события

**Актуальная web-реализация** (страницы create/edit, шапка, режимы BASIC/ADVANCED, единый блок «Основная информация»): [web-service-event-form.md](./web-service-event-form.md). Этот документ — продуктовое/визуальное описание концепции; детали вёрстки и файлов смотрите там.

# 1. Общая концепция

Окно добавления сервисного события в MotoTwin — это не обычная CRUD-форма.

Это полноценное рабочее пространство обслуживания мотоцикла:

- service-centered workflow;
- digital garage workstation;
- premium maintenance interface.

Интерфейс должен ощущаться как:

- Tesla service UI;
- профессиональный инструмент обслуживания техники;
- технический premium-продукт;
- интерфейс цифрового двойника мотоцикла.

Основная идея:

> пользователь работает не с полями формы, а с сервисным событием и набором узлов обслуживания.

# 2. Общая структура окна

Модальное окно крупное, полноразмерное.

## Размеры

Desktop:

- ширина: 1450–1500px;
- высота: 88–92vh;
- centered modal;
- внутренний scroll;
- затемненный overlay с blur.

## Общая структура

```text

Main Workspace
  Left Column
  Right Column
Sticky Footer
```

Каждая вкладка:

# 5. Main Workspace

Основное пространство разделено на две колонки.

## Левая колонка

Ширина:

- 38–40%.

Назначение:

> метаданные сервисного события.

Содержит:

1. Основная информация
2. Стоимость
3. Напоминание
4. Фото и документы
5. Предварительный итог

---

## Правая колонка

Ширина:

- 60–62%.

Назначение:

> workflow service-bundle.

Это главный визуальный фокус модального окна.

Содержит:

1. Узлы и работы
2. Node cards
3. Редактирование деталей
4. Итоги по bundle
5. Preview последствий сохранения

# 12. Fast Mode

Fast mode показывает:

- выбранные узлы;
- типы работ;
- статус bundle;
- стоимость.

Но:

- без SKU;
- без part rows;
- без детальной детализации.

---

## Верхний баннер

```text
Быстрый режим
Выбраны узлы и указана общая стоимость.
Статусы узлов будут обновлены.
```

---

## Строки узлов

Каждая строка:

- иконка;
- название узла;
- путь категории;
- тип работы;
- зеленый статус;
- action icon справа.

Пример:

```text
Масло двигателя
Двигатель > Смазка
Замена
```

---

## Визуал строк

- compact dark rows;
- subtle separators;
- высокая плотность информации.

---

## Подвал bundle

Внизу:

```text
Детали
Работа
Итого
```

Итог:

- оранжевый;
- выровнен вправо.
